## 环境变量

### App 子域（可选）

| 变量 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `NEXT_PUBLIC_APP_URL` | 否 | `https://app.myhealthdata.foundation` | 登录与管理中心链接 |

### 联系表单 / Data Hub（联系页必需）

| 变量 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `DATA_HUB_BASE_URL` | 否 | `https://tuutpi.online` | Data Hub 服务地址 |
| `DATA_HUB_APP_ID` | **是** | — | 应用 ID（`app_xxx`） |
| `DATA_HUB_API_KEY` | **是** | — | API Key（`dh_xxx`） |

也支持别名（与对接指南一致）：

- `VITE_DATA_HUB_BASE_URL`
- `VITE_DATA_HUB_APP_ID`
- `VITE_DATA_HUB_API_KEY`

本地配置：复制 `.env.example` 为 `.env.local` 并填入真实值。

### Vercel 配置步骤

1. Project → Settings → Environment Variables
2. 添加上述 `DATA_HUB_*` 三个变量
3. **Environment 勾选 Production（及 Preview 如需要）**
4. 保存后进入 Deployments → **Redeploy**

> 仅保存变量而不 Redeploy，API 仍会返回 503。

### 验证联系表单

部署完成后，提交表单或请求：

```bash
POST https://www.myhealthdata.foundation/api/contact-events
```

成功应返回 **HTTP 201**，响应含 `ctc_` 开头的事件 ID。

## 域名配置

| 域名 | 用途 |
|------|------|
| `www.myhealthdata.foundation` | 官网 canonical 域名 |
| `myhealthdata.foundation` | 301 重定向至 www |
| `app.myhealthdata.foundation` | 独立 App 项目 |

`vercel.json` 已将非 www 域名永久重定向至 www。

---

## Cookie 联动

App 登录成功后需设置：

| 属性 | 值 |
|------|-----|
| Cookie 名 | `mhdf_logged_in` |
| domain | `.myhealthdata.foundation` |

Navbar 据此显示「进入管理中心」。

## 前端改动与环境变量

修改首页、组件、样式等 **前端代码不影响** 已配置的 `DATA_HUB_*` 变量。

仅在以下情况需重新检查环境配置：

- 删除或改名环境变量
- 修改 `app/api/contact-events/route.ts` 中的变量名
- 将 Data Hub 调用改为前端直连（不推荐）
