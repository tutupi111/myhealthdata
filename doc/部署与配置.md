# 部署与配置

## 环境要求

- Node.js 20+
- npm

## 常用命令

```bash
# 本地开发（默认 http://localhost:3000）
npm run dev

# 生产构建
npm run build

# 启动生产服务
npm run start

# 代码检查
npm run lint
```

## 环境变量

| 变量 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `NEXT_PUBLIC_APP_URL` | 否 | `https://app.myhealthdata.foundation` | App 子域地址，用于生成登录与管理中心链接 |

`.env*` 文件已加入 `.gitignore`，仓库内无 `.env.example`。

## 域名配置

| 域名 | 用途 |
|------|------|
| `www.myhealthdata.foundation` | 官网 canonical 域名 |
| `myhealthdata.foundation` | 通过 `vercel.json` 301 重定向至 www |
| `app.myhealthdata.foundation` | 独立 App 项目（登录 / 业务） |

### Vercel 重定向

`vercel.json` 配置：访问 `myhealthdata.foundation`（无 www）时，永久重定向至 `https://www.myhealthdata.foundation/:path*`。

## 部署平台

推荐使用 **Vercel** 部署：

1. 关联 Git 仓库
2. 框架自动识别为 Next.js
3. 按需设置 `NEXT_PUBLIC_APP_URL`
4. 绑定自定义域名 `www.myhealthdata.foundation`

## Cookie 联动说明

为使 Navbar 正确显示登录状态，App 子域在登录成功后需设置：

| 属性 | 值 |
|------|-----|
| Cookie 名 | `mhdf_logged_in` |
| domain | `.myhealthdata.foundation` |
| 作用 | 官网读取后显示「进入管理中心」 |

若 App 未正确设置该 Cookie，官网 Navbar 将始终显示「登录」。

## 构建产物

本项目为 Next.js App Router 静态生成（SSG）为主：

- 18 个内容页面（2 语言 × 9 路径）预生成
- `sitemap.xml`、`robots.txt` 动态生成
- 无 Server-side API 路由
