# 项目概述

## 项目名称与定位

**myhealthdata** 是 MyDataHealth Foundation 的官方网站，面向公众宣传「健康数据属于每个人」的核心理念，展示基金会使命、项目与平台入口，并将用户引导至独立应用子域完成登录与业务操作。

| 项目 | 说明 |
|------|------|
| 名称 | myhealthdata |
| 版本 | 0.1.0 |
| 类型 | 纯前端静态营销/展示网站 |
| 生产域名 | https://www.myhealthdata.foundation |
| 应用子域 | https://app.myhealthdata.foundation |
| 许可证 | MIT |

## 技术栈

| 层级 | 技术 |
|------|------|
| 框架 | Next.js 16.1.6（App Router） |
| UI | React 19.2.3 + TypeScript 5 |
| 样式 | Tailwind CSS 4 |
| 字体 | Google Inter（`next/font`） |
| 部署 | Vercel |
| 国际化 | 自研静态翻译（`lib/i18n/translations.ts`），支持英文 / 中文 |

依赖极简：仅 `next`、`react`、`react-dom`，无状态管理库、无 HTTP 客户端、无 ORM。

## 双站点架构

```
┌─────────────────────────────────────┐
│  www.myhealthdata.foundation        │  ← 本仓库
│  营销官网 / 双语静态页 / SEO          │
│  读取 Cookie: mhdf_logged_in        │
└──────────────┬──────────────────────┘
               │ 外链登录 / 管理中心
               ▼
┌─────────────────────────────────────┐
│  app.myhealthdata.foundation        │  ← 独立项目
│  登录 / 仪表盘 / 实际业务功能        │
│  写入 Cookie: mhdf_logged_in        │
└─────────────────────────────────────┘
```

官网通过 Cookie 感知用户是否已在 App 登录：App 登录成功后需设置 `domain=.myhealthdata.foundation` 的 `mhdf_logged_in` Cookie，Navbar 才会显示「进入管理中心」。

## 目录结构

```
myhealthdata/
├── app/                    # Next.js App Router 页面
│   ├── [lang]/             # 多语言路由（en | zh）
│   ├── layout.tsx          # 根布局
│   ├── page.tsx            # 根路径重定向
│   ├── sitemap.ts          # 站点地图
│   └── robots.ts           # 爬虫规则
├── components/             # UI 组件（导航、页脚、卡片、背景动画等）
├── lib/
│   ├── config.ts           # 外部 App URL、Cookie、角色配置
│   └── i18n/               # 国际化文案与工具函数
├── public/                 # 静态资源
├── doc/                    # 项目文档（本目录）
├── package.json
├── vercel.json             # 域名重定向配置
└── next.config.ts
```

## 当前版本范围

v0.1.0 已完成 **官网壳层** 的全部核心能力：双语页面、营销落地页、门户引导页、SEO、响应式布局，以及与 App 子域的登录联动设计。

**不在本版本范围内**：用户注册/登录实现、医疗数据存储与处理、联系表单提交、CMS 内容管理、Analytics 集成等。
